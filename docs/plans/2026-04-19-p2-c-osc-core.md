# P2.C OSC Core

## Goal

Port the minimum parser-facing slice of `upstream/ghostty/src/terminal/osc.zig`
needed to unblock Phase 3 `Parser` work, without dragging in the broader OSC
surface reserved for Phase 4.

## Upstream files

- `upstream/ghostty/src/terminal/osc.zig`
- `upstream/ghostty/src/terminal/osc/parsers/change_window_title.zig`
- `upstream/ghostty/src/terminal/osc/parsers/change_window_icon.zig`
- `upstream/ghostty/src/terminal/osc/parsers/report_pwd.zig`
- `upstream/ghostty/src/terminal/osc/parsers/mouse_shape.zig`
- `upstream/ghostty/src/terminal/osc/parsers/color.zig`

## MoonBit target files

- `src/terminal/osc.mbt`
- `src/terminal/osc_wbtest.mbt`
- `docs/plans/2026-04-19-p2-c-osc-core.md`

## Dependencies and invariants

- Depends on `P2.0` contract notes in
  `docs/plans/2026-04-19-p2-parser-prereqs.md`.
- Preserve the dedicated OSC parser state machine shape:
  prefix-state parsing first, trailing-data capture second, parse on `end`.
- Preserve terminator tracking:
  - `BEL` => `Terminator::Bel`
  - `ESC \\` => `Terminator::St`, with parsing triggered by the `ESC` byte
    and the trailing `\\` consumed by the outer parser
- Keep fixed-buffer capture with the same effective size limit as the upstream
  fixed buffer path.
- Minimal Phase 2 command surface:
  - `invalid`
  - `change_window_title`
  - `change_window_icon`
  - `report_pwd`
  - `mouse_shape`
  - `color_operation` for OSC `4`, `5`, `10`-`19`, `104`, `110`-`119`
- Note on OSC `105`: `upstream/ghostty/src/terminal/osc/parsers/color.zig`
  supports `osc_105`, but `upstream/ghostty/src/terminal/osc.zig` does not
  route the parser state machine to `105`. Phase 2 follows the reachable
  upstream parser states and leaves `105` for a later review if upstream wiring
  changes.
- Phase 4 still owns the broader OSC surface:
  hyperlink, semantic prompt, clipboard, kitty protocols, iTerm2, rxvt,
  notifications, context signal, and text sizing.

## Acceptance criteria

- `OscParser` exposes `new`, `reset`, `deinit`, `next`, and `end`.
- `Command` and related color-operation types are sufficient for Phase 3
  parser integration.
- Direct OSC tests cover title parsing, UTF-8 title bytes, terminators, fixed
  buffer overflow, report-pwd / mouse-shape basics, and minimal color
  operations.
- `moon info`, `moon fmt`, `moon check`, and `moon test` pass.
