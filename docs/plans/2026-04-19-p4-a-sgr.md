# P4.A SGR

## Goal

Translate Ghostty's `sgr.zig` parser into MoonBit as a package-local semantic
decoder that preserves the upstream colon/semicolon behavior, typed attribute
output, and unknown/fallback semantics.

## Upstream files

- `upstream/ghostty/src/terminal/sgr.zig`

## MoonBit target files

- `src/terminal/sgr.mbt`
- `src/terminal/sgr_wbtest.mbt`
- `src/terminal/pkg.generated.mbti`
- `docs/plans/2026-04-19-p4-a-sgr.md`

## Dependency notes

- This task depends on the Phase 3 parser state machine and the Phase 4 decoder
  contract notes in `docs/plans/2026-04-19-p4-semantic-decoder-contracts.md`.
- `sgr.mbt` consumes parser-produced CSI parameter arrays and separator bits.
- The decoder stays package-local for now so Phase 7 can decide the final
  public API shape deliberately rather than leaking parser-internal semantic
  scaffolding early.

## Acceptance criteria

- SGR attribute parsing matches the upstream typed surface for:
  - unset
  - unknown(full, partial)
  - style toggles and resets
  - underline styles
  - 8-color, bright-color, 256-color, direct RGB color
  - underline-color forms and reset
- Colon-mode parsing is only accepted for `4`, `38`, `48`, and `58`.
- Unsupported or malformed colon runs degrade to `Unknown` rather than
  panicking.
- Direct-color colon and semicolon forms preserve upstream colorspace handling.
- Tests cover the parser behavior in the same task.
