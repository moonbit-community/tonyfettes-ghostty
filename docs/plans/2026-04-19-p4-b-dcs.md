## Goal

Translate the bounded Phase 4 DCS semantic handler from upstream Ghostty into
MoonBit within the parser package write set.

This task covers the handler lifecycle and the initial command surface only:

- unknown DCS -> ignore
- XTGETTCAP
- DECRQSS

Tmux control mode remains deferred for this slice to keep the task green and
within the owned files.

## Upstream Files

- `upstream/ghostty/src/terminal/dcs.zig`

## MoonBit Target Files

- `docs/plans/2026-04-19-p4-b-dcs.md`
- `src/terminal/dcs.mbt`
- `src/terminal/dcs_wbtest.mbt`
- `src/terminal/pkg.generated.mbti`

## Dependency Notes

- The local parser Phase 3 port already provides the raw DCS lifecycle and
  metadata needed here:
  - `DcsHook({ intermediates, params, final_byte })`
  - `DcsPut(byte)`
  - `DcsUnhook`
- This task intentionally stays above parser-core and below stream
  integration.
- Local `parser.mbt` keeps `Dcs` package-private. This handler lives in the
  same package and can consume it without widening parser visibility.
- Upstream tmux support depends on additional `tmux` parser/viewer modules and
  termio integration that do not exist locally yet, so tmux is explicitly
  deferred in this task.

## Acceptance Criteria

- Preserve the handler lifecycle shape:
  - `hook(dcs) -> ?Command`
  - `put(byte) -> ?Command`
  - `unhook() -> ?Command`
- Cover inactive, ignore, XTGETTCAP, and DECRQSS states.
- Keep the command surface to XTGETTCAP and DECRQSS for this slice.
- Keep the API narrow and avoid exposing parser-core internals publicly.
- Add focused white-box tests for:
  - unknown DCS
  - XTGETTCAP basics
  - XTGETTCAP multiple keys
  - XTGETTCAP mixed/invalid data preservation and uppercasing
  - DECRQSS valid requests
  - DECRQSS invalid/overflow requests
  - max-bytes overflow behavior

## Review Findings

- `DcsHandler`, `DcsCommand`, `XtGetTCapCommand`, and `DecrqssRequest` remain
  package-private, so this slice does not widen the public parser API.
- Lifecycle shape matches the upstream bounded contract for this slice:
  `hook(dcs) -> ?Command`, `put(byte) -> ?Command`, `unhook() -> ?Command`.
- Unknown DCS hooks and overflowed payloads enter ignore and unhook to `None`,
  matching upstream behavior for ignored/error states.
- XTGETTCAP uppercases buffered ASCII bytes on unhook and splits keys lazily
  via `next()`, without hex-decoding.
- DECRQSS preserves the upstream request classification surface for `m`, `r`,
  `s`, and ` q`, with other one- and two-byte payloads classified as
  unrecognized.
- `moon info` did not add any DCS symbols to `pkg.generated.mbti`. The file did
  not retain DCS symbols after the final interface review.
