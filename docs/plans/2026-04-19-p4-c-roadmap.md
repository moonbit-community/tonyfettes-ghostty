# P4.C OSC High-Frequency Roadmap

## Goal

Turn `P4.C` into bounded OSC parser slices instead of treating hyperlink,
semantic prompt, and the rest of the high-frequency parser surface as one broad
implementation task.

## Upstream scope

- `upstream/ghostty/src/terminal/osc.zig`
- `upstream/ghostty/src/terminal/osc/parsers/hyperlink.zig`
- `upstream/ghostty/src/terminal/osc/parsers/semantic_prompt.zig`
- any other already-translated high-frequency parser helpers needed by
  `osc.zig`

## Dependency notes

- `P4.C` extends the current `src/terminal/osc.mbt` core.
- `P5.B2` depends on these parser outputs becoming available as typed
  `osc.Command` variants.
- `semantic_prompt` is materially larger than `hyperlink`, so it should not be
  bundled into the first parser slice.

## Completion criteria for P4.C

`P4.C` is complete only when all of the following are true:

- all `P4.C*` sub-slices are `done`
- every translated high-frequency OSC parser output is represented by a typed
  `osc.Command` variant or intentionally left unimplemented with the difference
  documented
- latest `P4.C` code slice passes:
  - `moon check`
  - `moon test`
  - `moon coverage analyze`
  - `moon fmt`
  - `moon info`
- coverage residuals in touched files are either eliminated or explicitly
  accepted in the slice audit
- `.mbti` public-surface diffs are reviewed and intentional
