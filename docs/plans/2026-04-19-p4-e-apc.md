## Goal

Translate the upstream APC helper from `upstream/ghostty/src/terminal/apc.zig`
into a bounded MoonBit slice that unblocks future stream integration without
pulling in the full kitty graphics implementation.

## Upstream files

- `upstream/ghostty/src/terminal/apc.zig`

## MoonBit target files

- `src/terminal/apc.mbt`
- `src/terminal/apc_wbtest.mbt`
- `src/terminal/pkg.generated.mbti`

## Dependency notes

- Keep the write set inside the APC helper files only.
- Reuse local UTF-8 decoding patterns from the package for turning captured APC
  bytes into a `String`.
- Do not pull in the upstream kitty graphics parser stack for this task.
- The placeholder kitty APC parsing must stay typed so later stream work can
  distinguish recognized APC commands from ignored input.

## Acceptance criteria

- Preserve the lifecycle shape: `start()`, `feed(byte)`, `end() -> ?Command`.
- Preserve first-byte identification semantics for APC command selection.
- Preserve the identify/ignore split from upstream.
- Support unknown APC -> ignore.
- Support kitty graphics identification boundary only.
- Keep kitty graphics support lightweight and typed; no storage/rendering work.
- Add focused white-box tests for:
  - unknown APC
  - malformed kitty graphics APC returning no command
  - integer overflow/error cases returning no command
  - valid kitty graphics APC yielding typed APC command output
