# P5.A Execute / ESC / CSI Roadmap

## Goal

Turn `P5.A` from a broad umbrella into an explicit stream-dispatch workstream
with clear completion criteria and only one active semantic gap at a time.

## Upstream scope

- `upstream/ghostty/src/terminal/stream.zig`
  - `execute`
  - `escDispatch`
  - `csiDispatch`

This roadmap does **not** include:

- OSC semantic dispatch (`P5.B`)
- DCS / APC passthrough (`P5.C`)
- terminal application / state application (`Phase 6`)

## Current assessment

`P5.A` was initially planned too broadly. That was acceptable while the stream
surface was mostly empty, but it is no longer the right planning unit now that
most execute / ESC / CSI arms are translated and individually audited.

At this point, the broad row in `docs/plan.md` should be treated as a workstream
label, not as the execution unit. The execution unit is a bounded sub-slice
with its own audit file, validation run, and atomic commit.

## Completion criteria for P5.A

`P5.A` is complete only when all of the following are true:

- all bounded `P5.A*` semantic sub-slices are `done`
- upstream `execute`, `escDispatch`, and `csiDispatch` have no remaining
  untranslated typed action branches
- any remaining differences from upstream are documented as intentional
  no-op/logging differences
- latest `P5.A` code slice passes:
  - `moon check`
  - `moon test`
  - `moon coverage analyze`
  - `moon fmt`
  - `moon info`
- coverage residuals in touched files are either eliminated or explicitly
  accepted in the slice audit
- `.mbti` public-surface diffs are reviewed and intentional

## Planning rule going forward

Do not open another generic `P5.A` implementation turn.

Pick exactly one remaining `P5.A*` sub-slice, create or update its audit file,
land code + tests + validation in one green commit, then update this roadmap.

`P5.A` is now closed. Remaining Phase 5 implementation lanes are `P5.B` and
`P5.C`.
