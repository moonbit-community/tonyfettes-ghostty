# P4.D OSC Long-Tail Roadmap

## Goal

Turn `P4.D` from one broad "remaining OSC" bucket into bounded parser slices
that can each land green and line up with downstream `P5.B4+` dispatch work.

## Upstream scope

- `upstream/ghostty/src/terminal/osc.zig`
- `upstream/ghostty/src/terminal/osc/parsers/clipboard_operation.zig`
- `upstream/ghostty/src/terminal/osc/parsers/kitty_color.zig`
- `upstream/ghostty/src/terminal/osc/parsers/osc9.zig`
- `upstream/ghostty/src/terminal/osc/parsers/rxvt_extension.zig`
- `upstream/ghostty/src/terminal/osc/parsers/iterm2.zig`
- `upstream/ghostty/src/terminal/osc/parsers/kitty_clipboard_protocol.zig`
- `upstream/ghostty/src/terminal/osc/parsers/context_signal.zig`
- `upstream/ghostty/src/terminal/osc/parsers/kitty_text_sizing.zig`

## Dependency notes

- `P4.D` extends the current `src/terminal/osc.mbt` parser core after `P4.C`
  is done.
- `P5.B4`, `P5.B5`, and `P5.B6` consume `P4.D` outputs in groups that already
  exist in upstream `stream.zig`:
  - direct typed stream actions
  - typed logging-only callbacks
- `osc9.zig`, `rxvt_extension.zig`, and `iterm2.zig` are not parser-core
  "misc" features. They each emit concrete upstream `osc.Command` variants and
  need explicit ownership in the plan.

## Completion criteria for P4.D

`P4.D` is complete only when all of the following are true:

- all `P4.D*` sub-slices are `done`
- every upstream long-tail parser routed by `osc.zig` is either:
  - translated to a typed MoonBit `osc.Command` variant, or
  - intentionally documented as unreachable / deferred
- latest `P4.D` code slice passes:
  - `moon check`
  - `moon test`
  - `moon coverage analyze`
  - `moon fmt`
  - `moon info`
- coverage residuals in touched files are either eliminated or accepted in the
  corresponding slice audit
- `.mbti` public-surface diffs are reviewed and intentional
