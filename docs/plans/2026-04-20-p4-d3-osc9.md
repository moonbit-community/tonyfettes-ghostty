# P4.D3 OSC 9 Notification and ConEmu Parser Family

## Goal

Translate upstream `OSC 9` parsing into typed MoonBit command variants, wire
the family through the OSC state machine, and keep downstream stream behavior
faithful by dispatching only the already-supported compatibility commands while
leaving the remaining ConEmu outputs as explicit no-ops until `P5.B5`.

- `upstream/ghostty/src/terminal/osc.zig`
- `upstream/ghostty/src/terminal/osc/parsers/osc9.zig`

## MoonBit target files

- `src/terminal/osc9.mbt`

- `src/terminal/parser_wbtest.mbt`

- `src/terminal/stream_test.mbt`

## Dependency notes

- This slice extends the `osc.mbt` numeric-prefix state machine with `9`.
- `OSC 9` mixes true desktop-notification payloads, ConEmu-specific command
  families, and compatibility outputs that reuse existing parser/stream
  payloads such as `ReportPwd` and `SemanticPrompt`.
- Stream dispatch remains intentionally partial in this slice:
  `ReportPwd` and `SemanticPrompt` reuse the existing stream outputs, while the
  remaining `OSC 9` outputs stay explicit no-ops until `P5.B5`.

## Acceptance criteria

- `OscParser` recognizes `OSC 9` and returns typed `Command` variants for the
  desktop notification and ConEmu parser family.
- The translated parser preserves upstream fallback behavior for malformed
  ConEmu subcommands by returning a desktop notification with the original
  captured body.
- Parser integration and stream defer behavior are covered by tests, including
  the compatibility `ReportPwd` and semantic-prompt paths.
