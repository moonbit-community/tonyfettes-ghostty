# P4.D4 RXVT and iTerm2 Notification and Bridge Extensions

## Goal

Translate the long-tail `OSC 777` RXVT notification parser and the `OSC 1337`
iTerm2 bridge subset that already maps onto existing typed MoonBit command
variants, while keeping currently unsupported `OSC 1337` keys rejected until a
later phase needs them.

- `upstream/ghostty/src/terminal/osc.zig`
- `upstream/ghostty/src/terminal/osc/parsers/rxvt_extension.zig`
- `upstream/ghostty/src/terminal/osc/parsers/iterm2.zig`

## MoonBit target files

- `src/terminal/osc.mbt`
- `src/terminal/osc_rxvt_iterm2.mbt`
- `src/terminal/osc_wbtest.mbt`
- `src/terminal/parser_wbtest.mbt`
- `src/terminal/stream_test.mbt`

## Dependency notes

- This slice extends the `osc.mbt` numeric-prefix state machine with `777` and
  `1337`.
- `OSC 777` only emits desktop notifications in the upstream parser.
- For `OSC 1337`, this slice only ports the branches that already map to
  existing MoonBit command variants:
  - `Copy=...` -> `ClipboardContents`
  - `CurrentDir=...` -> `ReportPwd`
- Other iTerm2 keys remain rejected for now, which matches the current
  effective MoonBit behavior because they have no typed downstream consumer in
  this phase.

## Acceptance criteria

- `OscParser` recognizes `OSC 777` and emits desktop notifications for the
  supported RXVT `notify;title;body` form.
- `OscParser` recognizes `OSC 1337` and emits typed clipboard/report-pwd
  commands for the translated iTerm2 bridge forms.
- Unsupported or malformed RXVT/iTerm2 forms are rejected instead of leaking
  accidental typed commands.
- Parser integration and current stream behavior are covered by tests.
