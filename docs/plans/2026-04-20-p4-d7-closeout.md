- `osc9.zig` family:
  - desktop notifications -> `Command::ShowDesktopNotification`
  - ConEmu sleep -> `Command::ConEmuSleep`
  - ConEmu message box -> `Command::ConEmuShowMessageBox`
  - ConEmu tab title -> `Command::ConEmuChangeTabTitle`
  - ConEmu progress -> `Command::ConEmuProgressReport`
  - ConEmu wait input -> `Command::ConEmuWaitInput`
  - ConEmu gui macro -> `Command::ConEmuGuiMacro`
  - ConEmu run process -> `Command::ConEmuRunProcess`
  - ConEmu output env var -> `Command::ConEmuOutputEnvironmentVariable`
  - ConEmu xterm emulation -> `Command::ConEmuXtermEmulation`
  - ConEmu comment -> `Command::ConEmuComment`
  - compatibility pwd bridge -> `Command::ReportPwd`
  - compatibility semantic-prompt bridge -> `Command::SemanticPrompt`
- `kitty_color.zig` -> `Command::KittyColorProtocol`
- `clipboard_operation.zig` -> `Command::ClipboardContents`
- `kitty_text_sizing.zig` -> `Command::KittyTextSizing`
- `context_signal.zig` -> `Command::ContextSignal`
- `rxvt_extension.zig` notification parser -> `Command::ShowDesktopNotification`
- `iterm2.zig`:
  - copy bridge -> `Command::ClipboardContents`
  - current-dir bridge -> `Command::ReportPwd`
- `kitty_clipboard_protocol.zig` -> `Command::KittyClipboardProtocol`

There are no remaining upstream `osc.zig` long-tail parser routes without a
MoonBit command representation.

## Intentional bridges

Some upstream parser families intentionally reuse existing command variants
instead of minting one variant per protocol name:

- `OSC 9;9` bridges to `ReportPwd`
- `OSC 9;12` bridges to `SemanticPrompt(FreshLineNewPrompt)`
- `OSC 777 notify` bridges to `ShowDesktopNotification`
- `OSC 1337 Copy` bridges to `ClipboardContents`
- `OSC 1337 CurrentDir` bridges to `ReportPwd`

These are not gaps. They are the actual semantic outputs produced by the
translated MoonBit parser lane.

## Review findings

- No untracked long-tail OSC parser outputs remain in the current MoonBit
  `osc.Command` surface.
- The public API added during `P4.D` is intentional: opaque payload types plus
  supporting enums for kitty color, kitty clipboard, kitty text sizing, and
  context signalling.
- Several long-tail parser outputs are still intentionally silent in
  `stream.mbt`; that is a downstream Phase 5 integration concern, not a `P4.D`
  parser completeness gap.

`P4.D` is complete.

The next downstream work is the Phase 5 stream lane, with `P5.B4` through
`P5.B6` now fully unblocked from the parser side.
