# P5.C DCS And APC Passthrough Roadmap

## Goal

Complete the remaining `stream.zig` passthrough surface after `P5.A` and
`P5.B` by wiring parser-emitted DCS and APC lifecycle actions into typed
`StreamAction` outputs without widening the public API beyond what downstream
consumers actually need.

## Constraint notes

- APC passthrough is straightforward: upstream forwards start / put / end as
  byte-oriented stream callbacks.
- DCS passthrough is narrower but trickier because the current parser-side
  `Dcs` struct is package-private. Exposing it directly through `StreamAction`
  would leak parser internals into the public stream API.
- The MoonBit stream layer therefore needs a stream-facing DCS hook payload
  type with only the contract downstream consumers need:
  - intermediates
  - params
  - final byte
- That stream-facing DCS payload should stay accessor-based so callers can
  inspect the header without mutating internal parser storage.

## Planned slices

### P5.C1 APC passthrough actions

Wire parser APC lifecycle actions to typed stream actions:

- `ApcStart`
- `ApcPut(Byte)`
- `ApcEnd`

Acceptance:

- APC lifecycle ordering matches upstream
- stream tests prove passthrough actions emit before later printable text
- validation batch passes green in the same commit

### P5.C2 Stream-facing DCS hook payload

Introduce the public stream payload needed for DCS passthrough without exposing
the parser-private `Dcs` struct directly.

Acceptance:

- new payload type is scoped to stream use, not parser internals
- accessor surface is sufficient for blackbox tests and downstream consumers
- `.mbti` review confirms no unjustified parser internals leak

### P5.C3 DCS passthrough actions

Wire parser DCS lifecycle actions to typed stream actions:

- `DcsHook(<stream-facing payload>)`
- `DcsPut(Byte)`
- `DcsUnhook`

Acceptance:

- hook / put / unhook ordering matches upstream
- stream tests prove passthrough actions emit and later printable text resumes
- validation batch passes green in the same commit

### P5.C4 Closeout audit

Audit the final `StreamAction` surface and placeholder removals:

- no DCS / APC placeholders remain
- public API additions are intentional and minimal
- coverage / formatting / generated interface state is reviewed
