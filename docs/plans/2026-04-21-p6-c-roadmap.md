# P6.C Screen / Page / Hyperlink Roadmap

## Goal

Split the screen-facing terminal model work into green slices that can land
 before page editing and scrolling are fully translated.

## Upstream files

- `upstream/ghostty/src/terminal/Screen.zig`
- `upstream/ghostty/src/terminal/page.zig`
- `upstream/ghostty/src/terminal/Terminal.zig`

## Planned slices

### P6.C1 Hyperlink and semantic metadata state

Status:

- done

Scope:

- active OSC8 cursor hyperlink state
- semantic prompt metadata (`seen`, click mode, cursor semantic content)
- prompt-row markers needed by later screen mutation logic

### P6.C2 Screen title and auxiliary metadata

Status:

- todo

Scope:

- terminal title storage
- any small screen-adjacent metadata that later bridge/effect slices need

### P6.C3 Cell and row backing values

Status:

- todo

Scope:

- row prompt markers and row wrap flags
- cell content/style/hyperlink/semantic markers in MoonBit value form

### P6.C4 Editing and scrolling carriers

Status:

- todo

Scope:

- printable writes
- erase/insert/delete operations
- scroll-region mutation behavior
- DECALN and full-reset-facing state resets

## Review notes

- `P6.C1` intentionally stops before any row/cell storage so the semantic and
  hyperlink metadata can land without fake page allocators
- `P6.C3` and `P6.C4` should stay tightly coupled to the exact upstream
  mutation semantics, because `P6.1.C` depends on them for faithful action
  handling
- `P6.C1` is now landed; `P6.C2` is the next smallest screen-owned slice that
  can move the model forward without inventing placeholder row storage
