# Mini Tmux PTY Example

## Goal

Add a small native MoonBit example under `examples/tmux` that demonstrates a
minimal tmux-like multiplexer built with `moonbit-community/pty`,
`moonbit-community/tty`, and the translated Ghostty terminal surface. The demo
must not shell out to the system `tmux` binary.

## Accepted Design

- Add an executable package at `examples/tmux`.
- Add `moonbit-community/pty@0.2.0`, `moonbit-community/tty@0.2.4`, and
  `moonbitlang/async@0.19.1` to the examples module.
- Use `@tty.Tty` to own the outer terminal raw mode, alternate screen, key
  events, resize events, and redraws.
- Start with one shell pane backed by `@pty.Pty`.
- Add prefix-key commands:
  - `Ctrl-b c` creates a new shell pane.
  - `Ctrl-b n` and `Ctrl-b o` switch to the next pane.
  - `Ctrl-b p` switches to the previous pane.
  - `Ctrl-b b` sends a literal `Ctrl-b` to the active pane.
  - `Ctrl-b q` and `Ctrl-b x` quit the demo.
- Feed each pane's PTY output into its own `@terminal.StreamTerminal`.
- Forward ordinary keyboard input to the active pane's PTY.
- Render the panes with a simple native MoonBit layout and status line.
- Connect `StreamTerminal` query replies back to each pane's PTY through a small
  local reply buffer, because `StreamTerminal::write_pty` is synchronous while
  `Pty::write` is async.
- Keep the example package native-only; the existing Rabbita examples remain
  JS-only.

## Target Files and Surfaces

- `examples/moon.mod`
- `examples/tmux/moon.pkg`
- `examples/tmux/main.mbt`

No `src/terminal` implementation or public API change is expected.

## API / Interface Diff

- New package: `tonyfettes/ghostty-examples/tmux`
- New executable command: `moon -C examples run --target native tmux`
- `src/terminal/pkg.generated.mbti` should remain unchanged.

## Why Existing Code Cannot Be Reused As-Is

- `tools/stream_terminal_perf` feeds synthetic byte streams and does not own a
  PTY.
- `tools/terminal_playground_core` is a wasm-facing playground and only records
  terminal query replies; it cannot call async native PTY APIs.
- The Rabbita asciinema example replays recorded terminal output rather than
  spawning a live terminal program.
- The previous sketch that launched the system `tmux` binary demonstrated PTY
  capture, but it did not implement a multiplexer and depended on an external
  program.

## Open Questions

- Runtime success depends on a POSIX-like shell being available as `sh`; the
  example is native-only and Unix-oriented because it owns PTYs and a raw TTY.

## 2026-06-02 UI Checkpoint

### Goal

Make pane navigation visible in the mini tmux demo by rendering pane boundaries
and exposing the active pane cursor in the outer terminal.

### Accepted Design

- Keep the existing prefix commands unchanged.
- Render each visible pane as an ASCII box with a header on the top border,
  vertical content borders, and a bottom border used as the visual split line.
- Use `[*]` in the pane header for the active pane and `[ ]` for inactive panes.
- Resize each pane PTY to the box interior, so child shells see the same rows and
  columns that are actually available for content.
- After each redraw, map `StreamTerminal::cursor_x/y()` from the active pane into
  the pane's content area and move the outer tty cursor there when
  `StreamTerminal::cursor_visible()` is true.

without changing pane PTY contents.

### Accepted Design

- Keep all key bindings, pane layout, and PTY behavior unchanged.
